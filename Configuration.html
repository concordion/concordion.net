Configuring Concordion.NET

Concordion.NET provides several configuration options for the execution of automated acceptance tests.

With Concordion.NET 1.5.0 there is just one type of configuration files used. This configuration file is specific for each project containing Concordion.NET specifications and/or fixtures. This configuration file has the same name as the DLL file, but the file extension ".dll" is replaced by ".config". The <spec>.config file has to be located in the same directory as your <spec>.dll to be loaded by Concordion.NET.
