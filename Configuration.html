Configuring Concordion.NET

Concordion.NET uses two types of configuration files to define basic parameters for the execution of automated acceptance tests.

1. Project specific configuration

The first configuration file is specific for each project containing Concordion.NET specifications and/or fixtures. This configuration file has the same name as the DLL file, but the file extension ".dll" is replaced by ".config". The <spec>.config file has to be located in the same directory as your <spec>.dll to be loaded by Concordion.NET.

2. Concordion.config

This second configuration file contains setting for the test runners to be used. The default test runner of Concordion.NET is the RunTestRunner. An example configuration is shown below. The config file has to be located in the same directory as the Concordion.dll in order to be found by the Concordion.NET framework.

Runners

Runners are used to call and run one specification from another specification. Another specification may be in a different testing suite (like FIT, FitNesse, Cucumber, etc.)

Runner

Each runner element has two attributes that must be present to be recognized properly. They are "alias" and "type". The alias is the name of the runner used in the concordion:run statement. The type is the Assembly-Qualified Name of the type that you will associate with the alias and will be used to run the called specification. See the specification for Run for more details. 


<?xml version="1.0" encoding="utf-8" ?>
<Concordion>
  <Runners>
    <Runner
    	alias="runtestrunner"
    	type="Concordion.Spec.Concordion.Command.Run.RunTestRunner,
  		Concordion.Spec, Version=1.0.0.0,
  		Culture=neutral,
 	 	PublicKeyToken=null" />
  </Runners>
</Concordion>
